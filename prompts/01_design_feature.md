# Prompt Codex — Concevoir une fonctionnalité datapack

Aide-moi à concevoir une nouvelle fonctionnalité datapack pour Minecraft Java Edition 26.1.

Travail demandé :
1. Reformule le besoin gameplay.
2. Propose une architecture datapack claire :
   - namespaces ;
   - fonctions `load`, `tick`, utilitaires ;
   - scoreboards ;
   - predicates ;
   - advancements ;
   - item components ;
   - tags ;
   - damage types ;
   - enchantments éventuels.
3. Indique les alternatives techniques possibles.
4. Indique les limites et risques.
5. Propose un plan de fichiers.
6. Ne crée rien avant validation.

Règles :
- N’invente aucune syntaxe.
- Pour les commandes et data components récents, cite ou demande les sources.
- Propose d’abord une version minimale testable, puis une version propre/étendue.
- Prévois une fonction de debug désactivable.
